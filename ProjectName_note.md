## 0. 写作目标与证据状态

### 推荐写法

- 先记录问题形成、路线演化、方案取舍和证据状态，再从中提炼论文的引言、方法、算例和结论。
- 主动保留失败结果、被放弃路线、不成熟模型、未完成推导和当前局限，因为这些内容能帮助团队复核判断、减少重复试错。
- 使用克制、可验证的表述，例如“当前证据支持”“该判断仍需验证”“该路线已归档”“该结论目前仅在某算例中观察到”。
- AI 生成的段落、图注、表格说明和论文草稿应通过 `.cursor/rules/03-public-note-and-paper-skill-bridge.mdc` 的写作风格检查，并用 `.cursor/rules/lexicon.md` 做词表筛查。
- 信息缺失时直接标注 `【待补充】`、`【尚未验证】`、`【文献待核查】`、`【当前证据不足】`、`【Git 待提交】`、`【文档滞后】`、`【尚未实现】`、`【代码待核查】`、`【结果待复核】`，让后续工作一眼可见。

### 证据状态标签

对每个判断性结论标注证据状态：

| 状态 | 使用场景 |
| --- | --- |
| 已由理论推导支持 | 有清晰公式、假设和推导链，但不一定已有数值验证 |
| 已由数值实验支持 | 有登记过的 case、结果文件、指标和配置 |
| 已由代码实现支持 | 有可定位的函数、入口或测试，但结果可能尚未充分验证 |
| 已有 Git 版本记录 | 有 branch、commit、tag、issue 或 PR 可追溯 |
| 仅为当前假设 | 目前用于推进建模，但尚无充分证据 |
| 尚未验证 | 需要补推导、补实验、补代码或补文献 |
| 结果不稳定 | 不同参数、样本、算例或重复运行下表现不一致 |
| 当前证据不足 | 已有材料的支撑力度低于当前表述 |
| 已被实验排除 | 已有实验表明该判断不成立 |
| 暂时放弃 | 当前不进入主线，但保留复盘价值 |
| 文献待核查 | 需要补最近竞争者或理论出处 |
| 代码尚未实现 | 文档有方案，代码还没有 |
| 结果待复核 | 有输出，但尚未完成一致性和质量检查 |

### 5.5 基本模型

先写最小可验证模型：

$$
\min_{x,u,z} \quad F(x,u,z;\theta)
$$

subject to

$$
G_i(x,u,z;\theta) \le 0,\quad i=1,\ldots,m,
$$

$$
H_j(x,u,z;\theta) = 0,\quad j=1,\ldots,p.
$$

其中：

- $F(\cdot)$ 表示【待补充】。
- $G_i(\cdot)$ 表示【待补充】。
- $H_j(\cdot)$ 表示【待补充】。

若使用松弛、近似、线性化、矩匹配、场景法、分解或 SCA，应明确原始问题、变换后问题、等价或近似条件，以及近似误差如何影响最终 claim。

### 8.2 正式图输出要求

- 每个 case 的正式图放入 `result/<case>/figures/`。
- 生成绘图代码前，默认先用 `$powerlit-power-systems-paper-writing` 基于 PowerLit 近邻论文生成或修订 `01_IDEA/figure_plan.md`；本地 PowerLit 不可用时，再使用 `03_REFERENCE/`、用户提供文献或联网检索，并把 fallback 来源、图型和可借鉴的展示组织写入上方表格。
- 绘制参数图、灵敏度图或参数扫描图前，先查 SOTA 论文一般比较哪些参数、范围、步长、归一化方式、默认值和横轴组织；本文采用不同参数或范围时，在研究笔记、manifest 或 caption 草稿中说明原因。
- 新图建议先用 `ProjectName_utils.plotting.create_figure("single-column")` 创建图窗；默认 IEEE profile 使用 10 pt 字号，并把图宽限制在 IEEE 双栏论文的一栏宽度内。复杂小图可以显式使用 8 pt，但需要在 manifest 中记录。目标期刊给出更具体要求时，以目标期刊为准。
- 折线图和柱状图建议调用 `ProjectName_utils.plotting.place_legend(ax)`，默认把图例放在图内自动位置；正式使用前检查图例是否遮挡关键数据。
- 使用 `ProjectName_utils.plotting.save_figure` 导出 `.fig`、`.png`、`.svg`、绘图数据、`figure_manifest.jsonl` 和 `figure_check_report.md`；正式图 metadata 必须填写 `claimId`、`physicsReproduction` 和 `evidenceRole`，`evidenceRole` 取上表的四个取值之一。
- 保持原始数据含义稳定；筛选、变换、对数轴零值处理或异常点说明同步写入 manifest 和检查报告。
- 异常点压缩主体细节时，可以额外生成诊断图；原始全量图和诊断图并列保留，正文需要使用诊断图时同步说明理由。
- 组合展示先导出独立子图，例如 `Fig05a`、`Fig05b`、`Fig05c`，再进入论文排版。
- 同一方法或类别的颜色、线型、marker、灰度和 hatch 必须全局一致，且黑白出版可辨。默认图例先用中文短标签，标签本身应有明确物理含义；英文稿阶段再统一翻译。

### 8.3 图组叙事和方法标注

正式图默认对应论文算例分析或实验结果部分。物理复现图组不是原理说明图、机制设计图、算法流程图或理论框架图，而是在算例分析中用 `本文方法` 的模型输出复现真实测量、高保真仿真、解析参考、可信 benchmark 或已知物理约束下的实际物理行为。

| 图组类型 | 记录规则 |
| --- | --- |
| 算例介绍图 | 可先于结果图出现，说明测试系统、源荷时序、概率分布、场景集或样本特征 |
| 第一组结果图：物理复现 | 强制先登记；展示 `本文方法` 对算例物理实际的复现能力，例如状态量、约束边界、源荷时序响应、概率分布、空间趋势或可行运行状态 |
| 外部 SOTA / baseline 对比 | 可覆盖主指标、其他指标、跨 case 泛化、运行时间等；方法名用 `Pappu2017` 或 `AuthorYear` 这类来源标注，并登记对应文献 |
| 本文方法消融 | 只比较 `本文原始方法` 和 `去掉<物理模块或机制>`；写清楚在本文方法上去掉了什么 |

正常情况下，外部方法对比图组和本文方法消融图组分开组织。

### 8.4 图型边界

| 图型 | 规则 |
| --- | --- |
| 结果值对比图 | 同时显示基准解和待比较方法 |
| 误差图 | 不显示基准自身的零误差 |
| 跨数量级误差 | 使用对数纵轴，并记录零值或负值处理阈值 |
| 分布距离指标 | 按节点或逐点绘制；带物理量纲时标单位 |
| 概率密度对比图 | 以基准 histogram 为底，叠加各方法 PDF 曲线；每个代表点单独出图 |
| 参数图 | 先查 SOTA 常比较的参数、范围、默认值和横轴组织；本文参数选择不同于常用设置时写明原因 |
