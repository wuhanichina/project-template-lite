# project-template-lite Workspace Rule

This file is a Google Antigravity workspace rule. It is intentionally a thin
bootstrap; the detailed repository policy lives in `AGENTS.md` and
`.cursor/rules/`.

Before editing code, documentation, comments, generated prose, or template
structure:

1. Read `AGENTS.md`.
2. Read every file listed in `AGENTS.md`.
3. Treat `.cursor/rules/` as shared repository policy, not Cursor-only local
   state.
4. Keep human-facing research files focused on research content, evidence,
   model logic, and writing material. Put agent execution policy in rule files.
5. When changing agent compatibility, update `AGENTS.md`, `GEMINI.md`,
   `CLAUDE.md`, `.agents/rules/`, `.agent/rules/`, and `README.md` together.

Use positive, evidence-bound research wording: minimum identifiable research
object, concrete engineering scene, model relation, measurable output, and
evidence source.
