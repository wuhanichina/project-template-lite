# Claims

本文件登记可验证研究结论。`C1`、`C2`、`C3` 等编号用于连接算例入口、图表 metadata、证据索引和论文草稿；正文表述应使用正向技术语言，说明最小可辨识研究对象、具体工程对象、工程场景、本文处理方式、验证判据和证据状态。

正文表述使用领域通用术语、工程对象、物理量、模型关系和可查证算例名称。生造词、内部实验配置代号、临时参数组名、分支名或脚本缩写只写入证据编号、对应入口、原始结果位置、run manifest、Git 记录或配置字段。

写结论时使用：

```text
最小可辨识研究对象 + 具体工程对象 + 发现或验证的判断 + 本文模型/方法处理的物理关系 + 理论意义 + 工程价值 + 解决的工程问题 + 可验证输出
```

创新点主句直接说明本文面向哪个小同行研究对象，发现或验证了什么机制、规律、关系或研究猜想，建立了什么模型或计算流程，形成了什么理论意义，服务什么工程价值，解决了什么实际工程问题，输出了什么可验证结果。工程场景、证据口径、尚需验证内容和讨论材料分别写入表格字段；指标、算例和图表用于支撑上述判断。

| 状态 | 含义 |
| --- | --- |
| 待验证 | 已定义结论、对象和入口，等待推导、实现或实验 |

| 已由代码实现支持 | 有可定位的函数、入口或测试 |
| 结果待复核 | 已有输出，等待一致性和质量检查 |
| 当前证据不足 | 已有材料只支持较弱表述 |
| 已修正 | 结果促使结论表述、工程场景或证据状态发生调整 |
| 已归档 | 曾有参考价值，但不再支撑当前主线 |
